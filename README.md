*git 명령어

< git init >

현재 디렉토리에 git 저장소를 생성하는 명령어.

< git clone >

원격 저장소로부터 복제하는 명령어.

< git help >

명령어를 잊어버렸을 때 사용하는 명령어. 커맨드 라인에 명령어 입력시 21개의 가장 많이 사용하는 깃 명령어들이 나타남.

< git add 파일명 >

파일을 새로 추적하는 명령어. 

< git add * >

변경된 모든 파일 스테이징하는 명령어.

< git push origin master >

원격으로 보내는 명령어.

< git commit -a >

변경된 모든 파일을 커밋하는 명령어. 기본 에디터가 실행되면 커밋 메세지를 작성하고 저장한 후 종료한다.

< git commit -m "커밋 메세지" >

변경사항을 내 컴퓨터에 저장하는 명령어. 위 명령어를 실행하면 작업흐름상에 변경된 파일이 HEAD에 반영됨. 하지만, 원격 저장소에는 아직 반영이 되지는 않는다.

< git remote >

현재 원격 저장소의 이름을 알려주는 명령어.

< git remote update >

원격 저장소의 브랜치를 업데이트하는 명령어.

< git status >

 파일의 상태를 확인하는 명령어. 
 
< git diff >

스테이징 영역과 현재 작업트리의 차이점을 보여주는 명령어.

< git checkout --파일명 >

현재 브랜치를 파일명으로 변경한다.

< git full >

로컬저장소를 원격저장소에 맞게 갱신하기 위해 사용하는 명령어.

< git branch >

현재 존재하는 branch 조회하는 명령어.

< git branch 브런치명 >

branch를 만드는 명령어.

< git checkout >

활성 branch 변경하는 명령어.

< git branch -d 브런치명 >

branch 삭제하는 명령어.

< git merge 브런치명 >

branch명의 내용을 현재 branch로 합치는 명령어.

< git tag >

태그 목록을 볼 수 있는 명령어. 

< git tag 태그명 브랜치명 >

태그 붙이는 명령어.

< git rebase 브랜치명 >

해당 브랜치의 변경사항을 현재 브랜치에 적용하는 명령어.

< git diff >

Local과 Remote의 차이점 보여주는 명령어.

< 제목 Headers >
#으로 시작하는 텍스트.

#은 하나부터 여섯개까지 쓸 수 있고, #이 늘어날때마다 제목의 수준은 내려간다.

(보통 글씨 크기가 작아진다.)

	          
# h1                      
## h2
### h3
#### h4
##### h5
###### h6

또는 -, =을 이용하여 h1, h2를 쓸 수 있다.

실행결과

h1
===

h2
--
